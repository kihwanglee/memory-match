# PRD: MemoryMatch+ 카드 매칭 게임

## 1. 제품 개요

### 1.1 제품 비전
MemoryMatch+는 사용자의 기억력과 집중력을 향상시키는 웹 기반 카드 매칭 게임입니다. 단순한 카드 뒤집기를 넘어 다양한 난이도와 테마를 제공하여 모든 연령대의 사용자가 즐기면서 두뇌를 훈련할 수 있는 플랫폼을 목표로 합니다.

### 1.2 목표 사용자
- 주요 타겟: 기억력 향상을 원하는 성인 사용자 (25-55세)
- 부가 타겟: 두뇌 훈련에 관심 있는 시니어, 집중력 향상을 원하는 학생
- 사용 시나리오: 출퇴근 시간, 휴식 시간, 취침 전 가벼운 두뇌 활동

### 1.3 핵심 가치 제안
- **접근성**: 별도 설치 없이 웹 브라우저에서 즉시 플레이
- **점진적 난이도**: 쉬운 단계부터 시작하여 자연스럽게 실력 향상
- **지속적 동기부여**: 기록 추적과 성취 시스템으로 꾸준한 플레이 유도
- **개인화**: 선호하는 테마와 난이도 선택 가능

## 2. 핵심 기능 명세

### 2.1 게임 모드
#### 2.1.1 클래식 모드
- 같은 그림의 카드 2장을 찾아 매칭
- 난이도별 카드 수:
  - Easy: 4x3 (12장, 6쌍)
  - Medium: 4x4 (16장, 8쌍)
  - Hard: 6x4 (24장, 12쌍)
  - Expert: 6x6 (36장, 18쌍)

#### 2.1.2 타임 어택 모드
- 제한 시간 내 모든 카드 매칭
- 난이도별 제한 시간:
  - Easy: 90초
  - Medium: 120초
  - Hard: 180초
  - Expert: 240초
- 남은 시간에 따라 보너스 점수 부여

#### 2.1.3 무한 모드
- 시간 제한 없이 자유롭게 플레이
- 연습 및 휴식용 모드

### 2.2 테마 시스템
#### 2.2.1 기본 테마
- 동물: 귀여운 동물 일러스트
- 과일: 다채로운 과일 이미지
- 기하학: 심플한 도형 패턴
- 자연: 풍경과 자연물

#### 2.2.2 테마 특성
- 고해상도 이미지 또는 벡터 그래픽
- 색상 구분이 명확하여 식별 용이
- 문화적으로 중립적인 디자인

### 2.3 점수 및 기록 시스템
#### 2.3.1 점수 계산
- 기본 점수: 매칭 성공 시 100점
- 콤보 보너스: 연속 매칭 시 점수 가산 (2콤보: 110점, 3콤보: 120점...)
- 시간 보너스: 타임 어택 모드에서 남은 시간 1초당 10점
- 실수 페널티: 불일치 시 -5점

#### 2.3.2 통계 추적
- 게임별 기록:
  - 완료 시간
  - 시도 횟수
  - 정확도 (성공 매칭 / 전체 시도)
  - 획득 점수
- 누적 통계:
  - 총 플레이 시간
  - 총 게임 수
  - 평균 완료 시간
  - 최고 점수

### 2.4 진행 및 성취 시스템
#### 2.4.1 레벨 시스템
- 경험치 기반 레벨업
- 게임 완료 시 경험치 획득
- 레벨업 시 새로운 테마 또는 아이콘 언락

#### 2.4.2 업적 시스템
- 예시 업적:
  - "첫 발걸음": 첫 게임 완료
  - "완벽주의자": 한 번의 실수 없이 게임 완료
  - "번개 손놀림": Expert 난이도를 2분 이내 완료
  - "마라토너": 누적 100게임 완료
  - "콤보 마스터": 10콤보 달성

## 3. 사용자 경험 (UX) 설계

### 3.1 화면 구성
#### 3.1.1 메인 화면
- 게임 시작 버튼 (크고 눈에 띄게)
- 모드 선택 탭 (클래식/타임 어택/무한)
- 난이도 선택 버튼
- 테마 선택 드롭다운
- 내 기록 보기 버튼
- 설정 버튼

#### 3.1.2 게임 화면
- 상단 정보 바:
  - 현재 점수
  - 경과 시간 (또는 남은 시간)
  - 시도 횟수
  - 일시정지 버튼
- 중앙 카드 그리드
- 하단: 힌트 버튼 (선택사항)

#### 3.1.3 결과 화면
- 게임 결과 요약:
  - 최종 점수
  - 완료 시간
  - 정확도
  - 획득 경험치
- 새로운 업적 표시 (있을 경우)
- 액션 버튼:
  - 다시 하기
  - 메인으로
  - 공유하기

### 3.2 상호작용 디자인
#### 3.2.1 카드 애니메이션
- 카드 뒤집기: 3D 회전 효과 (0.3초)
- 매칭 성공: 부드러운 페이드아웃 또는 축하 효과
- 매칭 실패: 카드가 다시 뒤집힘 (0.5초 딜레이)

#### 3.2.2 피드백
- 사운드: 카드 뒤집기, 매칭 성공/실패, 게임 완료 효과음
- 진동: 모바일에서 매칭 성공 시 짧은 진동
- 시각적 피드백: 콤보 시 화면 상단에 텍스트 표시

### 3.3 접근성
- 키보드 네비게이션 지원
- 색맹 모드: 색상 외에 패턴이나 아이콘으로도 구분 가능
- 사운드 on/off 옵션
- 애니메이션 감소 옵션

## 4. 기술 요구사항

### 4.1 프론트엔드
- **프레임워크**: React 또는 Vue.js
- **상태 관리**: Redux 또는 Vuex
- **스타일링**: CSS-in-JS (styled-components) 또는 Tailwind CSS
- **애니메이션**: CSS transitions, Framer Motion

### 4.2 백엔드 (선택사항)
- 사용자 데이터 저장이 필요한 경우:
  - **서버**: Node.js + Express 또는 Firebase
  - **데이터베이스**: Firebase Firestore 또는 MongoDB
  - **인증**: Firebase Auth 또는 JWT

### 4.3 로컬 저장소
- 비회원 사용자를 위한 LocalStorage 활용
- 저장 데이터:
  - 게임 기록
  - 설정 (사운드, 테마 선호도)
  - 레벨 및 업적 진행 상황

### 4.4 반응형 디자인
- 데스크톱, 태블릿, 모바일 최적화
- 브레이크포인트:
  - Mobile: < 768px
  - Tablet: 768px - 1024px
  - Desktop: > 1024px

### 4.5 성능 요구사항
- 초기 로딩 시간: 3초 이내
- 카드 뒤집기 반응 시간: 100ms 이내
- 60fps 애니메이션 유지

## 5. 개발 로드맵

### Phase 1: MVP (4-6주)
- 클래식 모드 구현 (Easy, Medium 난이도)
- 기본 테마 2종
- 점수 및 시간 기록
- LocalStorage 기반 데이터 저장
- 반응형 UI (모바일/데스크톱)

### Phase 2: 기능 확장 (4주)
- Hard, Expert 난이도 추가
- 타임 어택 모드
- 테마 2종 추가
- 통계 화면
- 사운드 효과

### Phase 3: 고급 기능 (4-6주)
- 무한 모드
- 레벨 및 업적 시스템
- 소셜 공유 기능
- 리더보드 (백엔드 필요)
- 애니메이션 고도화

### Phase 4: 최적화 및 확장 (진행형)
- 성능 최적화
- A/B 테스팅
- 사용자 피드백 반영
- 추가 테마 및 모드

## 6. 성공 지표 (KPI)

### 6.1 사용자 참여
- DAU (Daily Active Users)
- 평균 세션 시간: 목표 10분 이상
- 게임당 평균 플레이 횟수: 목표 3회 이상
- 재방문율: 7일 내 목표 40% 이상

### 6.2 제품 성능
- 페이지 로드 시간: 3초 이내 유지율 95%
- 에러율: 1% 미만
- 모바일 반응성: 터치 반응 100ms 이내

### 6.3 사용자 만족도
- 평균 게임 완료율: 70% 이상
- 사용자 리텐션: 30일 목표 20%
- 앱 평점: 4.0/5.0 이상 (리뷰 기능 추가 시)

## 7. 위험 요소 및 대응

### 7.1 기술적 위험
- **위험**: 구형 브라우저 호환성 이슈
- **대응**: Babel, Polyfill 활용 및 브라우저 지원 범위 명확히 설정

### 7.2 사용자 경험 위험
- **위험**: 게임이 너무 쉽거나 어려워 이탈
- **대응**: 초기 난이도 조절 세심하게 테스트, 사용자 피드백 적극 수집

### 7.3 경쟁 위험
- **위험**: 유사한 게임이 많아 차별화 부족
- **대응**: 독특한 테마, 세련된 디자인, 성취 시스템으로 차별화
